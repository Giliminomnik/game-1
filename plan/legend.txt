"plan" ->

cliche.txt введён для ознакомления с тем, чего не должно быть(мы постараемся) в самом сюжете.

"Эскизы", DetectiveDiagrammPlot.xml(который открывается через draw.io) связаны между собой. В папке хранятся эскизы игры в тех её моментах, когда дело доходит до механики игры, которая описана в xml-файле.

ideas.txt служит идеями для игры, которые внесены на рассмотрение и обдумывание их реализации(неожиданно, не правда ли?)

problems.txt описывает проблемы, с которыми можно столкнуться с по ходу создания игры. Это в своём роде тоже список небольших клише. Следующим абзацем после описания проблемы пишется её предворительное решение. Уже утверждённые и проверенные решения будут помечаться в начале абзаца значком "+".

story.txt создан для описания вопросов, которые нужно прояснить для решения способа дальнейшего создания сюжета с помощью предложенных вариантов.

legend.txt ёбаная легенда, что здесь непонятного?
